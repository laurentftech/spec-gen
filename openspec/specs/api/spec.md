# API Specification

> Generated by spec-gen v1.0.0 on 2026-03-09 23:34

## Purpose

This document specifies the HTTP API exposed by the system.

## Requirements

### Requirement: APIAuthentication

The API SHALL require authentication via: Bearer JWT, None, API Key (ANTHROPIC_API_KEY or OPENAI_API_KEY)

#### Scenario: AuthenticatedRequest
- **GIVEN** a request with valid authentication credentials
- **WHEN** the request is processed
- **THEN** the request is authenticated successfully

#### Scenario: UnauthenticatedRequest
- **GIVEN** a request without authentication
- **WHEN** accessing a protected endpoint
- **THEN** the response status is 401 Unauthorized

### Requirement: Postopenspec

The API SHALL support `POST /api/generate` to generate openspec specification files from analysis results using llm.

**Request:**

```json
{
  "options": "GenerateApiOptions"
}
```

**Response:**

```json
{
  "result": "GenerateResult"
}
```

#### Scenario: GenerateOpenspec
- **GIVEN** Authenticated user with valid analysis results
- **WHEN** POST /api/generate is called with options
- **THEN** 200 OK with GenerateResult in response body

### Requirement: Getspecrequirement

The API SHALL support `GET /api/specs/requirements` to read spec requirements referenced in mapping.json.

**Request:**

```json
{
  "rootPath": "string"
}
```

**Response:**

```json
{
  "generatedAt": "string",
  "requirements": "Record<string, SpecRequirement>"
}
```

#### Scenario: GetSpecRequirements
- **GIVEN** A valid mapping.json file exists in the .spec-gen/analysis directory
- **WHEN** GET /api/specs/requirements is called
- **THEN** 200 OK with the generatedAt timestamp and requirements object in the response body

#### Scenario: GetSpecRequirementsWithMissingMapping.json
- **GIVEN** No mapping.json file exists in the .spec-gen/analysis directory
- **WHEN** GET /api/specs/requirements is called
- **THEN** 200 OK with an empty requirements object in the response body

#### Scenario: GetSpecRequirementsWithInvalidMapping.json
- **GIVEN** An invalid mapping.json file exists in the .spec-gen/analysis directory
- **WHEN** GET /api/specs/requirements is called
- **THEN** 200 OK with an empty requirements object in the response body

### Requirement: Postanalysis

The API SHALL support `POST /api/analyze` to run static analysis on the codebase.

**Request:**

```json
{
  "rootPath": "string",
  "maxFiles": "number",
  "excludePatterns": "string[]",
  "force": "boolean",
  "outputPath": "string",
  "onProgress": "function"
}
```

**Response:**

```json
{
  "repoMap": "RepositoryMap",
  "depGraph": "DependencyGraphResult",
  "artifacts": "object",
  "duration": "number"
}
```

#### Scenario: AnalyzeCodebase
- **GIVEN** Valid spec-gen configuration exists
- **WHEN** POST /api/analyze is called with valid parameters
- **THEN** 200 OK with analysis results in response body
- **AND** Analysis results include repository map, dependency graph, artifacts, and duration

#### Scenario: AnalyzeCodebaseWithExistingRecentAnalysis
- **GIVEN** Valid spec-gen configuration exists and recent analysis exists
- **WHEN** POST /api/analyze is called with force set to false
- **THEN** 200 OK with existing analysis results in response body
- **AND** Analysis results include repository map, dependency graph, artifacts, and duration

#### Scenario: AnalyzeCodebaseWithNoConfiguration
- **GIVEN** No spec-gen configuration exists
- **WHEN** POST /api/analyze is called
- **THEN** 500 Internal Server Error with error message in response body
- **AND** Error message indicates no spec-gen configuration found

```json
  "sslVerify": "boolean"

```

**Response:**

```json
    "filesVerified": "number",
    "filesPassed": "number",
    "filesFailed": "number",

  "duration": "number"
}
```

### Requirement: Functionspecgen

The API SHALL support `Function specGenRun(options: RunApiOptions = {})` to run the full spec-gen pipeline: init → analyze → generate.

**Request:**

```json
{
  "options": {
    "type": "object",
    "properties": {
      "projectPath": {
        "type": "string",
        "description": "Path to the project to analyze"
      },
      "configPath": {
        "type": "string",
        "description": "Path to the spec-gen configuration file"
      },
      "outputPath": {
        "type": "string",
        "description": "Path to the output directory"
      },
      "verbose": {
        "type": "boolean",
        "description": "Enable verbose logging"
      }
    }
  }
}
```

**Response:**

```json
{
  "type": "object",
  "properties": {
    "success": {
      "type": "boolean",
      "description": "Indicates if the spec-gen pipeline was successful"
    },
    "message": {
      "type": "string",
      "description": "A message describing the result of the spec-gen pipeline"
    },
    "data": {
      "type": "object",
      "description": "Additional data returned by the spec-gen pipeline"
    }
  }
}
```

#### Scenario: RunSpecGenPipeline
- **GIVEN** A valid project path and configuration
- **WHEN** The specGenRun function is called
- **THEN** The spec-gen pipeline is executed and a success message is returned

#### Scenario: RunSpecGenPipelineWithInvalidProjectPath
- **GIVEN** An invalid project path
- **WHEN** The specGenRun function is called
- **THEN** An error message is returned indicating the invalid project path
